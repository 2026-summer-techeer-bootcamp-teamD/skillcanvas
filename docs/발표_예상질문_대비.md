# 🎤 발표 예상 기술 질문 대비 (oh my skill)

> 사용법: 각 질문마다 **① 한마디(외워서 바로) → ② 풀어서 → ③ 더 파고들면** 순.
> 심사장에선 ①만 말해도 충분. ②③은 꼬리질문 대비용.

## ⭐ 가장 나올 확률 높은 5개 (이것만은 꼭)

### Q1. "GUI는 웹인데, 이걸 내 로컬(컴퓨터 파일·claude)이랑 어떻게 연결해? 그게 돼?"

> ⭐ **이게 이 프로젝트에서 제일 날카로운 질문.** 아래는 우리가 실제로 파고들며 도달한 결론 전체.
> 급하면 **한마디**만, 꼬리질문 들어오면 **여정 Q1-a~f** 순서대로 방어.

**① 한마디 (외워서 바로)**
> "웹 화면만 브라우저로 보고, 실제 파일·실행은 **내 컴퓨터에서 도는 작은 실행기**가 합니다. 웹사이트엔 그 실행기 **다운로드 버튼**을 달고요. 지금 이 PoC가 바로 그 실행기예요 — 보여드릴게요."

---

#### 🔬 심화 — 웹↔로컬 연결 완전정복 (실제 질의응답 여정)

**Q1-a. "일반 웹사이트는 내 파일 못 만지잖아. 어떻게 접근해?"**
- 일반 웹사이트: `내 브라우저 → 인터넷 → 남의 서버`. 남의 서버는 내 파일 **절대 못 만짐** (브라우저 보안).
- 우리 도구: `내 브라우저 → localhost → 내 컴퓨터에서 켜진 작은 프로그램`. 이 프로그램은 워드·VS Code처럼 **내 컴퓨터 앱이라 파일 접근 권한이 있음.**
- 핵심 분리: **화면(브라우저에 HTML)** 과 **일하는 놈(내 컴퓨터의 프로그램)** 이 따로. 브라우저는 "이 파일 저장해줘" **부탁만** 보내고, 실제 쓰기는 그 프로그램이 함.
- ⚠️ 흔한 오해: "클로드 계정 로그인하면 되나?" → **안 됨.** 파일은 클라우드가 아니라 내 컴퓨터에 있음 → 로그인이 아니라 **로컬 실행**이 답.

**Q1-b. "그건 PoC 얘기고. 배포하면 남들도 써야 하잖아. 웹이야 앱이야?"**
- **불편한 진실**: 로컬 파일을 만지는 도구는 **순수 호스팅 웹사이트가 될 수 없음.** 남의 서버에 올린 웹페이지가 접속자 파일을 만지면 = 해킹이라 브라우저가 원천 차단.
- → **로컬 파일을 만지려면, 반드시 사용자 컴퓨터에서 도는 뭔가가 있어야 함.** (우리만 그런 게 아니라 이 분야 전부)
- 그래서 배포가 **두 조각**으로 나뉨:
  - **파일 안 만지는 부분(갤러리·로그인·공유)** → 진짜 클라우드 웹사이트. 남들이 URL 접속. (Supabase)
  - **파일 만지는 부분(편집·실행)** → 사용자 컴퓨터에서 도는 로컬 실행기. 다운로드/설치.
- 근거: **Claude Code·n8n·Docker Desktop·Ollama** 전부 "로컬 프로그램 + 브라우저 화면" 구조. Dropbox·1Password도 "클라우드 계정 + 로컬 앱".

**Q1-c. "File System Access API 팝업으로 폴더 허용받으면? 앱 개발 안 해도 되잖아."**
- 맞음 — **파일 읽고 쓰기까진 순수 웹으로 됨** (브라우저가 "이 폴더 접근 허락?" 팝업 → 허락하면 JS가 직접 읽고 씀, Chrome/Edge).
- 됨: 파일 시각화 ✅ · 양방향 동기화 ✅ · config 내보내기 ✅
- **안 됨(경계선): 워크플로우 "실제 실행"** — 브라우저는 **프로그램(claude·MCP)을 못 띄움.** 이건 보안상 절대 안 열림.
- 결론: **"파일 만지기 + 인터넷 API 호출"까진 웹으로 OK, "내 컴퓨터에서 프로그램 실행"만 불가.**

**Q1-d. "그럼 실행도 원클릭으로 웹에서 → Claude Code까지 시키면 안 돼?"**
- 됨. 근데 조건: **순수 웹페이지 혼자선 claude를 못 띄우므로**, 사용자 컴퓨터에 claude를 대신 실행해줄 **작은 다리(=로컬 실행기)** 가 돌고 있어야 함.
- 흐름: `웹에서 [실행] 클릭 → 로컬 실행기가 claude 돌림 → 결과를 웹으로 반환`.
- **그 다리가 바로 우리 PoC(server.mjs).** 지금 "추천 받기" 누르면 실제 claude 돌아서 결과 뜨는 게 = 이 원클릭 실행 구조가 이미 작동 중.

**Q1-e. "Claude API를 사이트에 연동하면 로컬 조각 없이 다 되지 않아?"**
- **Claude API ≠ Claude Code.** API는 **클라우드의 두뇌**(말 걸면 텍스트로 답). 전화 상담 전문가 — 똑똑해도 **전화기 너머라 내 컴퓨터 파일·프로그램을 못 만짐.**
- API로 되는 것: **두뇌 = 자연어→추천·워크플로우 생성·노드 수정 ✅** (사이트에서 API 키로 호출, 로컬 claude 불필요 → 오히려 간단)
- API로 **안 되는 것**: 로컬 `.claude` 파일 읽고 쓰기 ❌ · 로컬 MCP·파일 실제 실행 ❌
- 결론: **API는 "두뇌"를 클라우드로 옮겨줄 뿐, "손(파일·실행)"은 여전히 로컬.** API 붙여도 로컬 조각은 안 사라짐.

**Q1-f. "claude는 앱만 아니라 CLI로도 부르잖아. 그걸로는? 그리고 '로컬 조각 만든다'가 대체 뭘 만든다는 거야?"**
- **CLI로 부르는 거 맞음.** 우리 PoC 실제 코드:
  ```js
  execFile('claude', ['-p', prompt], ...)   // = 터미널에 `claude -p "..."` 치는 것
  ```
  앱 API 안 쓰고 **CLI를 그대로 부름.**
- 함정: **브라우저는 이 CLI를 직접 못 침** (보안). 그래서 **누가 대신 쳐줘야 함 = 그게 로컬 조각.**
- **"로컬 조각"이 하는 일은 딱 3줄짜리 심부름:**
  ```
  ① 브라우저 요청 받기       ("이 자연어로 추천해줘")
  ② claude CLI 대신 쳐주기    execFile('claude', ['-p', ...])
  ③ 결과를 브라우저로 돌려주기 (화면에 카드로 뜸)
  ```
- 즉 **로컬 조각 = "브라우저 ↔ claude CLI 사이 심부름꾼".** 실제 생각·실행은 claude가 다 하고, 우리는 화면과 CLI를 이어주기만 함. → 무거운 실행 엔진이 **아님.** 이미 만든 PoC가 그거.
- 왜 브라우저는 안 되고 로컬 조각은 되나: 브라우저 안 코드 = 보안 감옥(샌드박스), 명령어 금지. 로컬 조각 = 내가 켠 프로그램, 감옥 밖, 명령어 칠 권한 있음.

**Q1-g. "그럼 웹사이트에 그 로컬 조각 다운로드 버튼 달면 되겠네?"** ✅ **정답 도달**
- 바로 그거. `웹 접속 → [로컬 실행기 다운로드] 클릭 → 한 번 설치 → 그 뒤론 웹에서 원클릭으로 다 돎`.
- 다운로드 2가지 맛: **설치파일(.dmg/.exe) 더블클릭** (비개발자 ← 우리 타겟) / **`npx skillcanvas` 명령어 복사** (개발자).
- 딱 하나 기억: 로컬 실행기는 **켜져 있어야** 웹과 통신. (트레이 상주/백그라운드로 만들면 사용자는 신경 안 씀. 웹이 "켜져 있나?" 확인 → 안 켜졌으면 "실행하세요" 안내.)
- 실제 사례: Docker Desktop·Ollama 다 "다운로드 버튼 하나" 방식.

#### 🏁 이 여정의 최종 결론 (한 그림)

```
[호스팅 웹사이트]                          [다운로드하는 로컬 실행기]           [Claude]
 · 갤러리·로그인·공유 (클라우드)             = "심부름꾼" 프로그램               · 두뇌: 자연어 이해·생성
 · 편집 화면(UI)                            ① 브라우저 요청 받기       ──API──▶  (클라우드, 텍스트 답)
 · [로컬 실행기 다운로드] 버튼               ② claude CLI 대신 실행     ──CLI──▶  · 실행: 파일·MCP 실제 작업
      │  다운/설치(1회)                     ③ 결과 웹으로 반환                    (로컬 Claude Code)
      └────────────────────────────▶  localhost 통신
```

> **역할 분담 한 줄**: **호스팅 웹**(갤러리·로그인·화면) + **다운로드 로컬 실행기**(파일·CLI 심부름) + **Claude**(두뇌·실행).
> 우리가 "만드는 것" = 무거운 엔진이 아니라 **① 화면 ② 파일 파서/직렬화 ③ 브라우저↔CLI 심부름꾼**. 실행은 claude가.

**꼬리질문 방어 요약**
- "웹이야 앱이야?" → **둘 다.** 갤러리는 웹, 편집·실행은 다운로드 로컬 실행기.
- "API 붙이면 로컬 필요없지?" → API는 두뇌만. **파일·실행은 로컬 필수.**
- "실행 엔진 만드는 거 무겁지 않아?" → 안 만듦. **claude CLI 심부름만.** PoC로 증명됨.
- "그게 진짜 돼?" → **지금 이 PoC가 그 구조.** 화면에서 보여주면 끝.

### Q2. "양방향 동기화가 진짜 돼? 파일 고치면 그림 바뀌고, 그림 고치면 파일 바뀌고?"

**① 한마디**
> "네, PoC에서 왕복 다 됩니다. 원리는 간단해요 — **파일을 읽어서 그림으로(파싱), 그림을 다시 파일로(직렬화)**. 정해진 형식이라 안정적으로 변환됩니다."

**② 풀어서**
- 스킬 파일(`SKILL.md`)은 형식이 정해져 있어요 (맨 위 설정부분 + 본문). 그걸 **읽어서 노드로 그리고**, 노드를 고치면 **다시 그 형식으로 써넣어요**.
- 핵심 안전장치: **본문은 절대 안 건드리고**(사용자가 쓴 자유 텍스트), 우리는 구조화된 부분(설정·MCP 연결·권한)만 수정. → 파일 깨질 위험 차단.
- 우리 PoC에 `본문 보존 ✓` 체크가 실제로 뜨는 이유가 이거예요.

**③ 더 파고들면**
- 이게 우리가 "직접 구현하는" 핵심 엔지니어링임 — 클로드가 대신 해주는 게 아니라, 우리가 만든 **미니 파서 + 직렬화기**.
- 실시간 감시(파일 바뀌면 자동 반영)는 확장 단계. 지금은 "다시 읽기" 버튼 방식.

### Q3. "실행 엔진 안 만든다며. 그럼 워크플로우는 뭐가 돌리는 거야?"

**① 한마디**
> "실행은 **이미 검증된 Claude Code가 합니다.** 우리는 실행 순서·연결을 짜주는 **오케스트레이터(지휘자)**고요. 바퀴를 새로 만드는 게 아니라, 있는 엔진을 잘 조립합니다."

**② 풀어서**
- 역할 분담이 핵심이에요:
  - **머리(판단·생성)** = 클로드
  - **손발(외부 작업: 노션 쓰기, 시트 기록)** = MCP
  - **지휘(순서·연결·config)** = 우리 도구
- "실행 엔진을 안 만든다"는 건 곧 **1명·2주에 현실적**이라는 뜻이기도 해요. 무모하게 런타임을 새로 안 짜니까.

**③ 더 파고들면**
- 완성된 워크플로우는 **config로 내보내서** Claude Code가 읽고 실행. 우리는 "악보를 그려주는 도구", 연주는 클로드.
- 무인/자동 트리거(사람 없이 스케줄 실행)는 클라우드 워커가 필요한 **확장 단계**. 지금은 "내 컴퓨터 켜져 있을 때 실행"까지가 범위.

### Q4. "자연어→워크플로우 자동생성, 그거 결국 클로드가 다 해주는 거 아냐? 너네가 만드는 기술이 뭔데?" ⚠️중요

**① 한마디**
> "클로드는 **이해·생성**을 하고, 우리는 그걸 **제약·검증·시스템 반영**합니다. 클로드 출력 그대로 쓰면 엉뚱한 노드가 나와요 — 우리가 만든 카탈로그·검증 규칙 안에서만 그리게 잡아주는 게 우리 몫입니다."

**② 풀어서** (반반이라고 솔직하게)
| 기술 | 누가 |
|------|------|
| 양방향 포맷 보존 동기화 | **100% 우리** (파서+직렬화기) |
| 자연어 → 구조 매핑 | **반반** — 이해/초안은 클로드, 제약·검증·에러복구·시스템반영은 우리 |
| 워크플로우 모델링+실행 | **대부분 우리** (DAG 구성, config 변환, 상태 관리) |
| MCP 카탈로그 | **우리 자산** (n8n 커넥터 같은 것) |

- 포인트: 클로드는 아무 노드나 만들어낼 수 있어요. 우리는 **"우리가 지원하는 MCP 카탈로그 안에서만, 검증된 형식으로만"** 나오게 통제해요.

**③ 더 파고들면 / 혹시 "클로드가 PoC도 만들었잖아?" 라고 하면**
- **"클로드로 개발한다"(개발 도구) ≠ "제품이 실행 중 클로드를 부른다"(아키텍처)** — 이 둘은 완전히 다른 얘기.
- v0·커서로 만든 앱을 "AI가 다 만들었으니 기술 아니다"라고 안 하죠. 도구로 뭘 쓰든, **우리가 설계한 시스템(동기화·검증·카탈로그·루프)이 결과물**이에요.

### Q5. "MCP 붙이려면 API 키·인증 필요하잖아. config만 만든다고 실행이 돼?"

**① 한마디**
> "맞아요, 키는 필요합니다. 우리가 만드는 config는 **'어떤 MCP를 어떻게 연결한다'는 설계도**고, 키 같은 비밀값은 사용자가 자기 환경에 넣습니다. 우리 서버가 키를 들고 있지 않아요 (보안상 오히려 안전)."

**② 풀어서**
- config 안엔 "노션 MCP를 이 명령으로 실행" 같은 **구조**만 있고, 실제 키는 `${NOTION_TOKEN}` 처럼 **자리표시자**로 둬요.
- 사용자가 자기 컴퓨터 환경변수에 키를 넣으면 → Claude Code가 그 키로 실제 연결.
- 키 없이 되는 MCP도 있어요 (예: Airbnb 검색 MCP) → 데모는 이런 걸로 잡으면 매끄러움.

**③ 더 파고들면**
- 무인 자동실행까지 가면 **비밀값 금고(secrets vault)**가 필요 — 이건 로드맵 3단계.
- 지금 범위: 사용자가 자기 키를 자기 환경에 넣고 로컬 실행.

## 🔸 추가로 나올 수 있는 것들

### Q6. "그냥 클로드한테 대화로 시키면 되잖아. 왜 이 도구가 필요해?"
> "대화는 **매번 다시 설명하고 끝나면 사라져요.** 우리는 **한 번 만든 걸 부품으로 저장·재사용**하고, **워크플로우로 반복 자동화**하고, **갤러리로 공유**합니다. 대화는 휘발되지만, 여긴 **자산으로 쌓여요.**"

### Q7. "n8n이랑 뭐가 달라? 이미 있는 거 아냐?"
> "n8n은 **데이터 흐름**을 그려요. 우리는 **당신의 Claude 스킬 자체**를 그리고, 그걸 **실제 로컬 파일과 양방향으로 동기화**합니다. n8n엔 없는 '내 AI 구성을 눈으로 보고 파일로 관리' 지점이에요."

### Q8. "갤러리에서 남의 워크플로우 가져오면 내 컴퓨터에 어떻게 들어와?"
> "갤러리엔 **가벼운 그래프 JSON만** 저장돼요. '가져오기' 하면 그 JSON을 **내 GUI 캔버스에 그대로 그려주고**, '내 것으로 저장' 하면 **기존 양방향 동기화가 내 `.claude`에 써줍니다.** 무거운 다운로드 로직 없이 이미 있는 기능 재사용이에요."

### Q9. "전체 실행 버튼이랑 배포 config 내보내기랑 뭐가 달라?"
> "**실행**은 지금 이 자리에서 테스트로 한 번 돌려보는 거고, **config 내보내기**는 그걸 **어디서든 반복 실행할 수 있는 설정 파일로 뽑는** 거예요. 실행=시연, config=배포용 포장."

### Q10. "config 내보내기, 무슨 원리로 되는 거야?"
> "워크플로우 그래프를 세 덩어리로 정리해서 내보내요:
> **① workflow spec**(어떤 노드가 어떤 순서로) + **② agent config**(각 AI 단계가 뭘 하는지) + **③ deployment config**(어떤 MCP·권한이 필요한지). 그래프를 읽어서 이 셋으로 변환하는 게 우리 코드예요."

### Q11. "비개발자 2명·주니어 1명인데 2주에 되겠어?"
> "실행 엔진을 안 만드는 게 핵심 전략이에요. **팀장이 코어(동기화·워크플로우)**, **주니어가 갤러리(관리형 로그인·저장, Supabase)**, **비개발 2명이 데이터·프롬프트·발표·QA**. 인증·DB는 맨손 구현 안 하고 관리형 서비스 씀."

### Q12. "트리거(자동 실행)는 돼?"
> "**내 컴퓨터 켜져 있을 때의 스케줄 실행**까지가 지금 범위예요. **사람 없이 24시간 무인 실행**은 클라우드 워커+비밀값 금고가 필요한 **로드맵 3단계**로 분리했어요. (범위를 솔직하게 자른 게 오히려 강점)"
