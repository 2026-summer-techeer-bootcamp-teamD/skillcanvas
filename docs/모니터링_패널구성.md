# SkillCanvas 모니터링 패널 구성

> Grafana(`localhost:3000`, admin/admin) **SkillCanvas** 폴더의 대시보드 5개 현재 상태 레퍼런스.
> 실제 JSON: `monitoring/grafana/provisioning/dashboards/json/*.json` (Prometheus datasource uid: `prometheus`)
> 갱신 시 이 문서도 같이 고칠 것 — 코드가 최종 근거이며, 이 문서는 그 스냅샷.

---

## 1. SkillCanvas - Overview (`skillcanvas-overview`)

서비스 상태 진입점. 상세 대시보드로 가는 drill-down 링크 포함(시간 범위 유지).

### 상단 Stat Row (항상 펼침, sparkline + threshold 색)

| 패널 | 쿼리 | 단위 |
| --- | --- | --- |
| RPS (요청/초) | `sum(rate(http_requests_total[$__rate_interval])) or vector(0)` | reqps |
| p95 응답 지연 (전체) | `histogram_quantile(0.95, sum by (le) (rate(http_request_duration_highr_seconds_bucket[$__rate_interval])))` | s |
| 5xx 에러율 (%) | `100 * (sum(rate(http_requests_total{status=~"5.."}[$__rate_interval])) or vector(0)) / clamp_min(sum(rate(http_requests_total[$__rate_interval])), 1e-9)` | percent |
| DB 커넥션 수 | `sum(pg_stat_database_numbackends) or vector(0)` | short |
| AI 호출률 (calls/s) | `sum(rate(ai_call_total[$__rate_interval])) or vector(0)` | cps |
| AI 실패율 (%) | `100 * (sum(rate(ai_call_total{status!="success"}[$__rate_interval])) or vector(0)) / clamp_min(sum(rate(ai_call_total[$__rate_interval])), 1e-9)` | percent |

### Row: API (기본 펼침)

| 패널 | 쿼리 |
| --- | --- |
| RPS (상태코드별, stacked) | `sum by (status) (rate(http_requests_total[$__rate_interval]))` |
| 응답 지연 p50/p95/p99 (전체) | `histogram_quantile({0.5,0.95,0.99}, sum by (le) (rate(http_request_duration_highr_seconds_bucket[$__rate_interval])))` |

### Row: AI (접힘)

| 패널 | 쿼리 |
| --- | --- |
| 모델별 호출률 (calls/s) | `sum by (model) (rate(ai_call_total[$__rate_interval]))` |
| 상태별 호출 분포 (calls/s, stacked) | `sum by (status) (rate(ai_call_total[$__rate_interval]))` |

### Row: Infra (Postgres) (접힘)

| 패널 | 쿼리 |
| --- | --- |
| Postgres TPS (commit+rollback) | `sum(rate(pg_stat_database_xact_commit{...}[$__rate_interval])) + sum(rate(pg_stat_database_xact_rollback{...}[$__rate_interval]))` |
| DB 커넥션 사용률 (%, Gauge) | `100 * sum(pg_stat_database_numbackends) / clamp_min(max(pg_settings_max_connections), 1)` |

두 패널 모두 Postgres 대시보드로 가는 drill-down 링크 포함(2026-07-17 추가).

### Row: Business (접힘)

| 패널 | 쿼리 |
| --- | --- |
| 시간당 증가율 (발행/생성/가져오기) | `sum(increase({workflow_published_total,skill_created_total,skill_imported_total}[1h]))` |

**아직 없음**: CPU/메모리(node-exporter/cAdvisor 미설치), template variable, deploy annotation. `docs/모니터링_개선_진행상황.md`의 "남은 일" 참고.

## 2. SkillCanvas - AI Call Metrics (`skillcanvas-ai-calls`)

`app/core/llm.py`의 `ask_claude()` 계측 기반. Claude 호출량·지연·실패율·토큰·비용.

| 패널 | 타입 | 쿼리 요지 |
| --- | --- | --- |
| AI 호출 총 건수 (선택 구간) | Stat | `sum(increase(ai_call_total[$__range]))` |
| AI 호출 실패율 (%, status별) | Stat | `sum by (status) (rate(ai_call_total{status!="success"}[5m])) / scalar(...)` |
| AI 호출 p95 소요시간 | Stat | `histogram_quantile(0.95, ...ai_call_duration_seconds_bucket...)` |
| 모델/상태별 호출 건수 (선택 구간) | Table | `sort_desc(sum by (model, status) (increase(ai_call_total[$__range])))` |
| 모델별 호출률 (calls/s) | Timeseries | `sum by (model) (rate(ai_call_total[5m]))` |
| 상태별 호출 분포 (calls/s, stacked) | Timeseries | `sum by (status) (rate(ai_call_total[5m]))` — error=빨강, parse_error=주황 |
| 모델별 호출 소요시간 p50/p95 | Timeseries | `histogram_quantile({0.5,0.95}, sum by (le, model) (...))` |

> AI 토큰/비용 계측(`ai_tokens_total`, `ai_cost_usd_total`)은 2026-07-16에 구현했으나 사용 여부가 불확실해 2026-07-17에 드롭함. 재도입 시 `docs/모니터링_개선_진행상황.md` 기록 참고.

## 3. SkillCanvas - 5xx by Handler (`skillcanvas-5xx-by-handler`)

`http_requests_total`(handler 라벨) 재사용, 코드 변경 없음.

| 패널 | 타입 | 쿼리 요지 |
| --- | --- | --- |
| 5xx 총 요청수 (선택 구간) | Stat | `sum(increase(http_requests_total{status=~"5.."}[$__range]))` |
| 5xx 에러율 (%) | Stat | `100 * sum(rate(...5..[5m])) / sum(rate(...[5m]))` |
| 핸들러별 5xx 누적 건수 | Table | `sort_desc(sum by (handler, method, status) (increase(...)))` |
| 핸들러별 5xx 요청률 (req/s) | Timeseries | `sum by (handler) (rate(http_requests_total{status=~"5.."}[5m]))` |
| 전체 요청 대비 상태코드 분포 (stacked) | Timeseries | `sum by (status) (rate(http_requests_total[5m]))` — 5xx=빨강 |

**참고**: 이 4개 패널은 `or vector(0)` 미적용 — 5xx가 0건이면 "No data"로 뜬다(계획서 2단계 미반영 상태).

`workflow_published_total`(공개 발행만 집계) / `skill_created_total` / `skill_imported_total`.

| 패널 | 타입 | 쿼리 요지 |
| --- | --- | --- |
| 워크플로우 발행 수 (선택 구간) | Stat | `sum(increase(workflow_published_total[$__range]))` |
| 스킬 생성 수 (선택 구간) | Stat | `sum(increase(skill_created_total[$__range]))` |
| 스킬 가져오기 수 (선택 구간) | Stat | `sum(increase(skill_imported_total[$__range]))` |
| 누적 추이 (3종) | Timeseries | 카운터 raw 값 3개 |
| 시간당 증가율 (건/h) | Timeseries, bars | `sum(increase({...}[1h]))` |

## 5. SkillCanvas - Postgres (`skillcanvas-postgres`, 2026-07-17 추가)

postgres-exporter(prometheuscommunity/postgres-exporter) 계측 기반. Grafana 커뮤니티 대시보드 ID `9628`(PostgreSQL Database)의 구성을 참고했으나, 원본은 Helm chart 전제(`release` 라벨)·구버전 Angular 패널(singlestat)이라 이 프로젝트의 `grafana/grafana:latest`에 그대로 임포트하면 대부분 깨짐 — 실행 중인 postgres-exporter의 `/metrics`를 직접 확인해 실제 존재하는 메트릭/레이블로 재작성함.

| 패널 | 타입 | 쿼리 요지 |
| --- | --- | --- |
| 서비스 상태 (선택 구간) | Stat | `pg_up or vector(0)` — 0/1을 DOWN/UP으로 매핑 |
| DB 크기 (skillcanvas) | Stat | `pg_database_size_bytes{datname="skillcanvas"}` |
| 활성 커넥션 수 | Stat | `sum(pg_stat_database_numbackends{datname!~"template.*\|postgres"})` |
| 커넥션 사용률 (%) | Gauge | `100 * sum(pg_stat_database_numbackends) / clamp_min(max(pg_settings_max_connections), 1)` |
| 캐시 히트율 (%) | Stat | `100 * blks_hit / (blks_hit + blks_read)` |
| Deadlock 발생 수 (선택 구간) | Stat | `sum(increase(pg_stat_database_deadlocks[$__range]))` |
| TPS (commit/rollback, DB별) | Timeseries | `rate(pg_stat_database_xact_commit\|xact_rollback[$__rate_interval])` |
| Tuple 처리량 (fetch/insert/update/delete) | Timeseries, stacked | `rate(pg_stat_database_tup_*[$__rate_interval])` |
| 세션 상태별 분포 | Timeseries, stacked | `sum by (state) (pg_stat_activity_count)` |
| 락 모드별 건수 | Timeseries, stacked | `sum by (mode) (pg_locks_count)` |
| 최장 트랜잭션 소요시간 | Timeseries | `max(pg_stat_activity_max_tx_duration)` |
| 테이블별 Dead Tuple 비율 (%) | Timeseries | `100 * n_dead_tup / (n_live_tup + n_dead_tup)` |
| 테이블별 Scan 비율 (Seq vs Index) | Timeseries | `rate(pg_stat_user_tables_seq_scan\|idx_scan[$__rate_interval])` |
| Autovacuum/Autoanalyze 실행 건수 | Timeseries, bars | `increase(pg_stat_user_tables_auto{vacuum,analyze}_count[$__rate_interval])` |
| WAL 크기 | Timeseries | `pg_wal_size_bytes` |
| Checkpoint 발생률 (timed vs requested) | Timeseries | `rate(pg_stat_bgwriter_checkpoints_{timed,req}_total[$__rate_interval]) * 3600` |

**검증**: 23개 PromQL 쿼리 전부 실행 중인 Prometheus(`localhost:9090`)에 대해 문법·실행 성공 확인(node 스크립트). 임시 Grafana 컨테이너(사용자 실행 중인 컨테이너는 미접촉, `skillcanvas_default` 네트워크에 연결해 실제 Prometheus 데이터로 확인 후 제거)로 5개 대시보드 전체 프로비저닝 성공 확인.

**원본(ID 9628) 대비 뺀 것**: CPU/메모리/파일디스크립터(`process_*` 메트릭, `release` 라벨 필요 — 우리 postgres-exporter엔 없음), `$instance`/`$datname`/`$mode` template variable(우선순위표 6번 항목에서 전체 대시보드 통합 예정).

## 메트릭 → 대시보드 매핑 (빠른 참조)

| 메트릭 | 정의 위치 | 쓰이는 대시보드 |
| --- | --- | --- |
| `http_requests_total`, `http_request_duration_highr_seconds` | `prometheus-fastapi-instrumentator` (main.py) | Overview, 5xx by Handler |
| `ai_call_total`, `ai_call_duration_seconds` | `backend/app/core/llm.py` | Overview, AI Call Metrics |
| `pg_stat_database_*`, `pg_settings_max_connections` | postgres-exporter | Overview, Postgres |
| `pg_stat_activity_*`, `pg_locks_count`, `pg_stat_user_tables_*`, `pg_stat_bgwriter_*`, `pg_wal_size_bytes`, `pg_database_size_bytes`, `pg_up` | postgres-exporter | Postgres |
| `workflow_published_total`, `skill_created_total`, `skill_imported_total` | `routers/workflows.py`, `routers/skills.py` | Overview, Business Metrics |
