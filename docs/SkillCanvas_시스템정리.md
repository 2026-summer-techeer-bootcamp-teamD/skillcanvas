# 🧩 우리 시스템 한눈에 정리 (업데이트본)

> **"말로 만드는 나만의 AI 업무 비서 빌더"**
> 제품은 범용, 발표 데모만 CS로 좁힘. PoC 내용을 그대로 살리고, **에이전트 + 하네스**가 새로 얹힌 버전.

*(작성일 2026-07-04)*

---

## 1. 한 줄 정의

- **제품(범용)**: Claude를 구독했지만 제대로 못 쓰는 사람이, **자기 업무를 말로 설명하면** → 스킬·MCP·에이전트를 조립해 **믿고 맡길 수 있는 AI 업무 비서**를 만들어주는 시스템.
- **발표 데모(좁힘)**: **CS 신입**을 주인공으로 여정을 보여줌.

---

## 2. 핵심 프레임 — "제품 ≠ 데모"

| | 제품 | 발표 데모 |
|---|------|----------|
| 범위 | **범용** (누구나 자기 업무 비서) | **CS 신입 하나** (구체) |
| 왜 | 아키텍처·비전 유지 | 멘토 "구체적으로" 충족 |
| 별(star) | **우리 시스템**(조립·하네스) | CS는 시스템을 보여줄 무대 |

> "주제를 좁힌다"가 아니라 **"발표 시나리오 대상을 정한다"**. 엔진은 범용, CS 경로만 완전히 배선.

---

## 3. 우리 도구의 "부품" = 스킬 + MCP + 에이전트

PoC는 **스킬 + MCP** 였고, 여기에 **에이전트**가 새로 추가됨.

| 부품 | 뭐냐 | 예 (CS) |
|------|------|---------|
| **스킬(Skill)** | 재사용 능력 정의 (프롬프트) | "컴플레인 대응 스킬" |
| **MCP** | 외부 도구 연결 (손) | Gmail, 택배조회 |
| **에이전트(Agent)** ⭐신규 | 스킬+도구로 **판단·생성하는 Claude 노드** | 긴급도 판단, 사과문 생성 |

> 에이전트 = 워크플로우 안에서 도는 **작은 Claude 두뇌**. MCP도 쓸 수 있음.

---

## 4. 전체 흐름 — 두 단계

### 【A】 조립 (자연어 → 그래프)
```
자연어로 업무 설명
   → (백엔드) Claude가 구조 생성  ※ 우리 카탈로그 안 MCP만 추천·불러오기
   → (우리) 검증: 카탈로그에 있나 / 그래프 유효한가
   → React Flow로 노드 렌더 → 사용자가 보고 수정
   → 저장: 스킬을 .claude에, MCP를 .mcp.json에 (양방향 동기화)
```
- **Claude = 조립 두뇌**, **우리 = 카탈로그 제약·검증·직렬화**

### 【B】 실행 (로컬 실행기 + 하네스)
```
로컬 실행기가 그래프를 순서대로 돌림
   → 에이전트 노드는 claude 호출, MCP 노드는 외부 도구 실행
   → 사이사이 하네스가 신뢰성 보장 (중복체크·검증·승인게이트)
```

---

## 5. 하네스 = 우리가 만드는 오케스트레이션 (핵심 백엔드)

> **에이전트·MCP = Claude Code 영역** (판단·외부작업, 재구현 X).
> **하네스 = 100% 우리 코드** → "config GUI가 아니다"의 증거 = 백엔드 어필.

**코어 3종 (2주 범위)**

| 하네스 | 하는 일 | CS 예 |
|--------|---------|-------|
| **① 중복체크 (메모리)** | 이미 처리한 건 제외 | "이 고객 어제 대응함 → 스킵" |
| **② 검증 (가드레일)** | 결과가 진짜인지 확인, 아니면 중단 | "조회결과가 진짜 배송지연 맞나" |
| **③ 승인 게이트 (사람 확인)** ⭐ | 실행 멈추고 → 사용자에게 물어봄 → 승인 시 재개 | "🚨 사과+쿠폰 발송할까요?" |

**로드맵 (지금 안 함)**: 조건분기, 서브에이전트, 병렬, 재시도 정책 등.
> 조건분기 제외 이유: 엔진 복잡도↑ 대비 데모 payoff 낮음. **검증 노드의 "안 맞으면 중단"**이 필요한 제어를 커버.

### ③ 승인 게이트 = 제일 어렵고 빛나는 엔지니어링
```
실행기: "승인 필요" 만나면
  1. run 상태를 로컬 SQLite에 저장 (어디까지 왔는지 + 중간 데이터)
  2. status="대기중" → 프론트에 팝업
  3. 실행 멈춤
        ⋯ 사용자가 [승인] 누를 때까지 대기 ⋯
프론트: [승인] → POST /approve
실행기: 저장된 상태에서 이어서 재개
```
→ **"중단 가능한 실행 + 상태 저장 + 재개"** = 진짜 하네스 엔지니어링.

## 6. 부품 명단 / 아키텍처

| 부품 | 역할 | 어디서 |
|------|------|--------|
| 프론트 (React+ReactFlow) | 화면·노드캔버스·승인팝업 | Vercel |
| 호스팅 백엔드 (FastAPI) | 갤러리·**카탈로그**·인증검증·조립용 AI호출 | EC2+Docker+Nginx |
| **로컬 실행기** (FastAPI+APScheduler) | 파일 read/write·**워크플로우 엔진**·**하네스**·Claude 호출 | 사용자 PC |
| Claude (CLI/API) | 두뇌 (조립 + 에이전트 노드) | Claude Code |
| MCP | 외부 손발 (Gmail·택배) | Claude Code가 구동 |
| **DB — 갤러리 (RDS)** | 유저·워크플로우·스킬·태그·좋아요 (**공유 데이터**) | AWS RDS (서버) |
| **DB — 실행메모리 (SQLite)** | 처리이력·실행상태(중단/재개)·스케줄 (**개인 데이터**) | 로컬 (사용자 PC) |
| .claude 파일 | 스킬·MCP 설정 | 로컬 |

### 데이터 위치 — "어디 저장?" (심사 대비 필수)

| 데이터 | 어디 | 왜 |
|--------|------|-----|
| 유저·워크플로우·스킬·태그·좋아요 | **RDS (서버)** | 공유·갤러리 |
| 처리 이력 (중복체크) | **로컬 SQLite** | 개인·프라이버시 |
| 실행 상태 (승인 중단/재개) | **로컬 SQLite** | 개인·런타임 |
| 컴플레인/메일 원본 | **저장 안 함** (Gmail서 그때그때 읽음) | 프라이버시 |

> 원칙: **로컬 실행기 = 개인/로컬 데이터, 서버 = 갤러리(공유)만.** 고객 메일 데이터는 우리 서버에 절대 안 올라감 → 프라이버시 어필.

## 7. CS 데모 실행 흐름 (태그본)

```
① [실행기·스케줄]  매일 9시 시작                    ← 트리거
② [MCP: Gmail]     컴플레인 메일 읽기               ← MCP (인바운드)
③ [에이전트]       긴급도 판단 "화남 95%, 배송이슈"  ← Claude 두뇌
④ [하네스: 중복]    로컬 SQLite 조회 → 어제 대응함? 스킵/진행 ← 우리 코드
⑤ [MCP: 택배]      배송 조회 "대전 HUB 체류중"       ← MCP (외부 API)
⑥ [하네스: 검증]    진짜 배송지연 맞나 → 통과해야 진행  ← 우리 코드
⑦ [에이전트]       사과+쿠폰 문구 생성               ← Claude 두뇌
⑧ [하네스: 🚨승인]  "발송할까요?" 팝업 → 대기 → 승인   ← 우리 코드 (하이라이트)
⑨ [MCP: Gmail]     사과+쿠폰 메일 발송              ← MCP (아웃바운드)
⑩ [하네스: 기록]    로컬 SQLite에 처리완료 저장 (다음날 중복방지) ← 우리 코드
```

## 8. PoC에서 살리는 것 vs 새로 만드는 것

| | 내용 |
|---|------|
| ✅ **PoC에서 살림** | 자연어→스킬/MCP 추천, .claude 양방향 동기화, 워크플로우 자동생성, 노드 편집, config 내보내기, MCP 카탈로그 |
| 🆕 **새로 추가** | **에이전트 노드**, **하네스 3종**(중복·검증·승인), 로컬 실행기의 **중단/재개 실행 엔진**, CS 특화 연동(Gmail·택배) |

---

## 9. 한 줄 결론

> **에이전트·MCP는 Claude가, 하네스(중복·검증·승인)는 우리가.**
> 우리가 만드는 것 = **"자연어로 조립되고, 카탈로그 안에서 검증되고, 사람 승인받으며 믿고 도는" 오케스트레이션 하네스.**
> 이게 "그냥 클로드한테 말하지 왜?"에 대한 답이자, config GUI와 우리를 가르는 지점.
