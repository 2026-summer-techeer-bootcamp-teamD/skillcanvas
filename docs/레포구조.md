## 2. backend/ 상세 — API 개발자는 여기만 보면 됨

```
backend/
├── app/
│   ├── main.py         라우터 등록 (새 API 만들면 여기 1줄 추가)
│   ├── core/
│   │   ├── config.py   설정(.env 읽기)
│   │   ├── db.py       DB 연결·세션  → Depends(get_db)
│   │   └── deps.py     인증          → Depends(get_current_user)
│   ├── models/         🟢 DB 테이블(ERD) — 완성됨, 갖다 쓰기만
│   │   ├── user.py  workflow.py  skill.py
│   │   ├── master_tag.py  skill_tag.py  workflow_tag.py
│   │   ├── tool_catalog.py
│   │   └── mixins.py   (created_at/updated_at 공통)
│   ├── schemas/        🟡 요청/응답 모양(Pydantic) — 여기에 추가
│   │   └── workflow.py    ← 복제용 예시
│   ├── routers/        🔴 API 엔드포인트 — 여기에 추가!
│   │   ├── workflows.py   ← ⭐ 복제용 템플릿 (6개 완성)
│   │   └── health.py
│   └── seed_tools.py   tool_catalog 초기데이터 (python -m app.seed_tools)
├── alembic/            마이그레이션 (모델 바꿀 때만)
├── requirements.txt    파이썬 패키지
└── pyproject.toml      black/ruff 설정
```

## 3. API 개발 = 이 3곳만 건드린다

| 순서 | 폴더/파일 | 하는 일 |
|------|-----------|---------|
| 1 | **`app/routers/`** | 엔드포인트 함수 (`GET`/`POST`/`PATCH`/`DELETE`) |
| 2 | **`app/schemas/`** | 요청·응답 모양(Pydantic) 정의 |
| 3 | **`app/main.py`** | `include_router(...)` 한 줄로 등록 |

> `models/`(테이블)·`core/`(DB·인증)는 **완성돼 있음** → import 해서 갖다 쓰기만.

## 4. 새 API 만드는 법 (예: 스킬)

**핵심: `workflows.py`를 복제한다.** (모든 CRUD + import 패턴이 거기 다 있음)

```
1) app/routers/workflows.py  →  복사  →  app/routers/skills.py
2) app/schemas/workflow.py   →  복사  →  app/schemas/skill.py
3) 안에서 이름 치환:
     Workflow → Skill,   WorkflowTag → SkillTag
     graph_json → content_md
     WORKFLOW_* → SKILL_*   (에러코드)
4) app/main.py 에 등록:
     from app.routers import skills
     app.include_router(skills.router, prefix="/api/v1")
5) docs/SkillCanvas_API명세서.md 의 해당 도메인 보고 세부 맞추기
```

**AI한테 시킬 때**: "`routers/workflows.py` 패턴을 그대로 따르고, `docs/SkillCanvas_컨벤션.md`를 지켜서 만들어줘."

### 도메인별 담당 (API명세서 기준)
- `users` (프로필) · `skills` · `tags` · `tool_catalog` · `assemble/recommend`(Claude 호출)
- 전부 `workflows.py` 복제 패턴으로 가능 (assemble류만 DB 없이 Claude 호출)

## 5. 자주 쓰는 명령어 (backend 폴더, venv 켠 상태)

```bash
docker compose up                    # (루트에서) 로컬 Postgres 켜기
alembic upgrade head                 # DB 테이블 생성/갱신
python -m app.seed_tools             # tool_catalog 초기데이터
uvicorn app.main:app --reload        # 서버 실행 → localhost:8000/docs
black app && ruff check app          # 커밋 전 검사 (CI에서도 검사함)

# 모델(테이블) 바꿨을 때만:
alembic revision --autogenerate -m "변경 설명"
alembic upgrade head
```

## 7. 브랜치 · PR 흐름

```
develop 에서  →  feature/기능이름  브랜치 파기
   →  개발 · 커밋 · push
   →  PR 열기 (대상: develop)
   →  팀장 승인 + CI(ruff·black) 통과
   →  팀장이 squash 머지 (브랜치 자동삭제)
```

- **직접 push·머지 불가** — 무조건 PR + 팀장 승인
- 브랜치 이름: `feature/스킬-발행`, `fix/xxx`, `docs/xxx` 식
- PR 제목: `[FEAT] 스킬 발행 API` 처럼 (이슈/PR 템플릿 자동으로 뜸)
