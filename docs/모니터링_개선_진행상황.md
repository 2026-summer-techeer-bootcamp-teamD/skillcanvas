### [완료] "유령 0 시리즈" 버그 수정 (2026-07-17)

`sum by (라벨) (...) or vector(0)` 처럼 **라벨이 남아있는 쿼리에 `or vector(0)`을 붙이면**, 실제 데이터가 있을 때도 라벨 없는 빈 시리즈(`{}`, 값 0)가 결과에 하나 더 추가됨(PromQL `or`는 라벨셋이 다르면 별개 시리즈로 취급). `sum(...)`처럼 `by` 없이 전체를 하나로 뭉개는 집계는 문제없음.

**적용한 결정**: 라벨별 분해 패널은 `or vector(0)` 제거(데이터 없으면 정직하게 "No data"). 단일값 stat/gauge/합계 패널만 `or vector(0)` 유지. `skillcanvas-node.json`의 raw 메트릭(Load Average 등)은 `avg()`로 감싸서 라벨을 지운 뒤 `or vector(0)` 적용.

**수정한 파일/패널**:
- `skillcanvas-ai-calls.json`: id2(실패율 status별), id4(테이블 model+status), id5(모델별 호출률), id6(상태별 호출 분포), id7(모델별 p50/p95) — `or vector(0)` 제거
- `skillcanvas-5xx-by-handler.json`: id3(테이블 handler+method+status), id4(핸들러별 요청률), id5(상태코드 분포) — `or vector(0)` 제거
- `skillcanvas-postgres.json`: id15(테이블별 Dead Tuple 비율)만 해당 — `or vector(0)` 제거(id11/id12/id16은 애초에 `or vector(0)`이 없어 문제 없었음, 이전 세션 감사 목록이 부정확했음)
- `skillcanvas-node.json`: id14/15(컨테이너별 CPU/메모리, by name) — `or vector(0)` 제거. id2/3/4/9(메모리 사용률, 루트 파일시스템 사용률, Load Average, 메모리 breakdown 3-target)는 `avg()`로 감싸서 라벨 제거 후 `or vector(0)` 유지
- 검증: 수정한 PromQL 15개 전부 실행 중인 `skillcanvas-prometheus`(읽기 전용 쿼리 API)에 문법·실행 성공 확인. `ruff`/`black` 통과.

## [완료] "No data" → 0 처리 전체 적용 (계획서 우선순위 2, 2026-07-17)

- `skillcanvas-ai-calls.json`(7개 패널/8개 target), `skillcanvas-5xx-by-handler.json`(5개), `skillcanvas-business-metrics.json`(5개 패널/9개 target)에 남아있던 원본 패널 전부에 `or vector(0)` 적용
- **부수 발견 및 수정**: `100 * A / clamp_min(B, 1e-9)` 형태의 비율 패널은 `B`(분모) 메트릭 자체가 완전히 비어 있으면(트래픽이 아예 없는 경우 등) `clamp_min`이 빈 벡터에 값을 채워주지 못해 여전히 "No data"가 뜨는 갭이 있었음 — 이미 커밋된 `skillcanvas-overview.json`(5xx 에러율, AI 실패율, DB 커넥션 사용률)과 방금 만든 `skillcanvas-postgres.json`(캐시 히트율, 커넥션 사용률, 테이블별 Dead Tuple 비율)에도 동일 패턴이 있어서 전부 `clamp_min((B or vector(0)), ...)` 형태로 같이 수정
- 검증: 5개 대시보드 전체 PromQL(17+23+8+5+9=62개) 실행 성공 확인 + 실제로 "No data"가 나던 5xx 에러율 쿼리가 수정 후 `0`을 반환하는 것 직접 확인. 임시 Grafana 컨테이너로 5개 대시보드 재프로비저닝 확인 후 정리
