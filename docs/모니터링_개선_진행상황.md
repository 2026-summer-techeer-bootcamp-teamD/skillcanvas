- `monitoring/grafana/provisioning/dashboards/json/skillcanvas-postgres.json` 신규 추가(패널 16개 + row 4개): 상단 Stat 6개(서비스 상태/DB 크기/활성 커넥션/커넥션 사용률/캐시 히트율/Deadlock) → Transactions(기본 펼침) → Sessions & Locks(접힘) → Tables(접힘) → Maintenance & WAL(접힘)
- Overview 대시보드 Infra row(Postgres TPS, DB 커넥션 사용률 패널)에 이 대시보드로 가는 drill-down 링크 추가, Overview 최상단 links에도 등록
- 검증: 23개 PromQL 쿼리 전부 실행 중인 Prometheus에 문법·실행 성공 확인(node 스크립트로 자동화). 임시 Grafana 컨테이너(`skillcanvas_default` 네트워크에 연결, 사용자 실행 중인 컨테이너는 미접촉)로 5개 대시보드 전체 프로비저닝 성공 + `pg_up`/캐시 히트율 등 실제 데이터 조회 확인 후 컨테이너 제거
- 원본(ID 9628) 대비 뺀 것: CPU/메모리/파일디스크립터(`process_*`, `release` 라벨 필요), template variable(`$instance`/`$datname`/`$mode` — 우선순위 6번에서 전체 대시보드 통합 예정)

## [완료] "No data" → 0 처리 전체 적용 (계획서 우선순위 2, 2026-07-17)

- `skillcanvas-ai-calls.json`(7개 패널/8개 target), `skillcanvas-5xx-by-handler.json`(5개), `skillcanvas-business-metrics.json`(5개 패널/9개 target)에 남아있던 원본 패널 전부에 `or vector(0)` 적용
- **부수 발견 및 수정**: `100 * A / clamp_min(B, 1e-9)` 형태의 비율 패널은 `B`(분모) 메트릭 자체가 완전히 비어 있으면(트래픽이 아예 없는 경우 등) `clamp_min`이 빈 벡터에 값을 채워주지 못해 여전히 "No data"가 뜨는 갭이 있었음 — 이미 커밋된 `skillcanvas-overview.json`(5xx 에러율, AI 실패율, DB 커넥션 사용률)과 방금 만든 `skillcanvas-postgres.json`(캐시 히트율, 커넥션 사용률, 테이블별 Dead Tuple 비율)에도 동일 패턴이 있어서 전부 `clamp_min((B or vector(0)), ...)` 형태로 같이 수정
- 검증: 5개 대시보드 전체 PromQL(17+23+8+5+9=62개) 실행 성공 확인 + 실제로 "No data"가 나던 5xx 에러율 쿼리가 수정 후 `0`을 반환하는 것 직접 확인. 임시 Grafana 컨테이너로 5개 대시보드 재프로비저닝 확인 후 정리
