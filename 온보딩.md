# 🚀 팀원 온보딩 가이드 (개발 시작 전 이것만!)

> 초보자용. 순서대로 따라 하면 됩니다. 막히면 팀장한테 물어보세요.

---

## 1단계. 설치 (한 번만)

| 프로그램 | 버전 | 받는 곳 |
|---------|------|--------|
| **Python** | 3.12 | python.org (또는 PyCharm이 받아줌) |
| **Node.js** | LTS 22 | nodejs.org |
| **Docker Desktop** | 최신 | docker.com (Postgres 돌리는 용도) |
| **PyCharm** | 최신 | jetbrains.com (Community 무료 OK) |
| **Git** | 최신 | git-scm.com |

> ※ **PostgreSQL은 설치 안 함** — Docker가 대신 돌려줍니다.
> ※ Windows는 Docker Desktop 설치 시 **WSL2**가 필요해요 (설치 마법사가 안내).

---

## 2단계. 레포 받기

```bash
git clone <레포 주소>
cd skillcanvas
```

---

## 3단계. 백엔드 세팅 (PyCharm)

1. PyCharm에서 **`backend` 폴더**를 프로젝트로 열기
2. PyCharm이 **"가상환경(venv) 만들까요?"** → **예** (Python 3.12 선택)
3. `requirements.txt` 뜨면서 **"설치할까요?"** → **예** (라이브러리 자동 설치)
4. **`.env` 파일 만들기**: `backend/.env.example` 복사 → `backend/.env` 로 이름 바꾸고, **팀장이 준 값** 채우기
   - `DATABASE_URL` 은 그대로 (로컬 Docker 주소)
   - `CLERK_...` 키는 팀장이 공유

---

## 4단계. DB 켜기 (Docker)

프로젝트 루트(`skillcanvas/`)에서:
```bash
docker compose up
```
→ Postgres 컨테이너가 뜹니다. (터미널 그대로 두고 다음 단계)

---

## 5단계. 백엔드 실행 & 확인

PyCharm 터미널(backend 폴더, venv 켜진 상태)에서:
```bash
uvicorn app.main:app --reload
```
브라우저에서 **http://localhost:8000/docs** 열기 → **Swagger** 화면

- `GET /api/v1/health` 실행 → `{"status":"ok","db_connected":true}` 나오면 **성공! 환경 세팅 끝.** ✅
- `db_connected: false` 면 → 4단계 Docker가 안 떠 있는 것

---

## 5.5단계. DB 테이블 만들기 (마이그레이션) + 시드

backend 폴더(venv)에서:
```bash
alembic upgrade head        # ERD의 7개 테이블 생성 (모델 → DB)
python -m app.seed_tools    # tool_catalog 초기 도구(Gmail·택배 등) 넣기
```
- Swagger에서 `GET /api/v1/workflows` 가 `200` + `{"items":[],"total":0,...}` 나오면 DB 연결·테이블 OK ✅
- **모델을 바꿨을 때**(컬럼 추가 등): `alembic revision --autogenerate -m "설명"` → `alembic upgrade head`

---

## 6단계. (프론트 담당만) 프론트 실행

```bash
cd frontend
npm install
npm run dev
```
→ http://localhost:5173 에 캔버스 뜨면 성공.

---

## 이제 개발 시작

- **모델**: ✅ 완료 (`backend/app/models/` — ERD 7개 테이블). 새 컬럼 필요하면 팀장과 상의 후 마이그레이션.
- **API 개발**: `docs/SkillCanvas_API명세서.md` 보고 `backend/app/routers/` 에 엔드포인트 추가
  - **📌 예시 템플릿 = `app/routers/workflows.py`** (6개 엔드포인트 완성본). **이걸 복제**해서 스킬·태그 등 만들면 됨.
    - 스킬 = workflows 복제 후 `Workflow→Skill`, `graph_json→content_md`, `WORKFLOW_*→SKILL_*` 로 치환
  - 스키마(요청/응답)는 `app/schemas/`, 인증은 `Depends(get_current_user)` (`app/core/deps.py`)
- **규칙**: `docs/SkillCanvas_컨벤션.md` 꼭 지키기 (AI한테도 "이 컨벤션 지켜" 하고 시키세요)
  - 커밋 전 `black app && ruff check app` 통과시키기 (CI에서도 검사)

## 자주 나는 문제
- `docker compose up` 에러 → Docker Desktop이 실행 중인지 확인
- `db_connected: false` → Docker(4단계) 켜져 있는지 확인
- `uvicorn: command not found` → venv 활성화 확인 (PyCharm이 자동으로 해줌)
